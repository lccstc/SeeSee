SeeSee 报价墙 MVP 计划
Summary
P1 先做“客人原始报价行情墙”，不做万能解析、不做 AI 主链路、不做供应商自动问价。系统自动监听客人群消息，按客人群模板解析长报价单和信息完整的单条报价，生成带历史切片的标准报价数据，并展示“当前同卡最高价墙”。低置信、新格式、缺上下文短回复只进入异常区，不自动入墙。

Key Changes
新增统一报价结构：来源客人群、卡种、国家/币种、面额范围、倍数限制、卡片形态、价格、状态、限制备注、消息时间、生效时间、失效时间、原文链路、模板版本、解析置信度。
新增报价来源口径：一个客人群就是一个报价来源；同一客人群同一 卡种 + 国家/币种 + 面额 + 形态 的新价按消息时间覆盖旧价，旧价保留为历史。
新增脚本模板解析：先支持客人 1/2/3 三类样例模板；通用字典只做标准化，例如 Steam/Razer/Xbox/Apple、USD/US/美国、卡图/代码/横白卡/竖卡。
新增自动入墙规则：模板高置信解析出的完整价格自动生效；新卡种新增；缺卡种的短回复如 uk 10 6.25 先记录为异常，不自动生效。
新增行情墙页面：第一屏按卡种聚合展示当前最高价，字段包含卡种、国家/币种、面额/条件、形态、最高价、客人群、更新时间、状态、限制；点开可看所有客人报价排名和原文。
新增异常区：展示模板不匹配、新卡种/新国家、价格突变、无法确认状态、缺上下文短报价，供后续人工修正或模板升级。
保留后续路线：P2 先做“价格加错”检测；再逐步做重复加单、漏边、漏加、错群、双边公式成立但业务错；P3 才引入 AI 辅助异常解释和新模板生成；P4 做供应商机器人自动报价和缺价/过期问价。
Interfaces / Data
数据库增加报价文档表：保存整份客人群报价原文、来源群、发送人、消息时间、模板版本、解析状态。
数据库增加报价价格表：保存标准化报价行和历史有效区间，支持按交易发生时间回查当时价格。
数据库增加报价异常表：保存未自动入墙的解析异常、原文行、原因、建议处理状态。
API 增加：
GET /api/quotes/board：当前最高价墙。
GET /api/quotes/history：按卡种/国家/面额/形态查看历史价格。
GET /api/quotes/exceptions：查看异常报价。
POST /api/quotes/exceptions/resolve：后续用于人工修正或忽略异常。
页面增加：
/quotes：行情墙。
/quotes/exceptions：异常区，MVP 可先合并到同一页。
Test Plan
用客人 1/2/3 的真实样例做解析测试：确认 Apple、Steam、Razer、Xbox 等主流板块能落成标准报价行。
覆盖条件字段：卡图、纸质、电子、代码、横白卡、竖卡、5X/50X/100X、面额范围。
覆盖覆盖规则：同一客人群同一 SKU 新价格覆盖旧价格，旧价格保留历史失效时间；新增卡种不会清空旧卡种。
覆盖异常规则：问、不拿、待定、缺卡种短回复、模板不匹配、价格突变进入异常区。
覆盖行情墙排序：同一卡种/国家/面额/形态按当前有效价格从高到低，最高价显示在第一屏。
覆盖历史回查：给定交易时间点能查到当时有效报价，为 P2 错价检测做准备。
Assumptions
P1 以客人原始报价为事实源，不使用同事整理报价单参与最高价排名。
P1 不做每行人工审核，只有异常才需要人工介入。
P1 不让 AI 参与主链路；AI 仅作为后续异常解释和模板辅助能力。
P1 不做供应商机器人自动问价；但价格历史模型要为后续“缺价/超过 30 分钟自动问价”预留。
第一版只需要先跑通 2-3 个核心客人群模板，看真实效果后再扩展模板和异常处理。
