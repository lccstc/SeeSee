SeeSee 报价墙 MVP PRD
1. 产品目标
SeeSee 报价墙 MVP 的目标是把客人群里的原始报价自动沉淀成一面可查询、可排序、可追溯的“当前最高价行情墙”。

第一版不追求万能解析、不引入 AI 主链路、不做供应商自动报价。核心只解决一个问题：

同事能快速看到：现在某个卡种、国家、面额、形态下，哪个客人价格最高，价格是什么，多久前更新，有什么限制。

2. MVP 范围
P1 只做客人原始报价侧：

自动监听客人群消息。
按客人群模板解析长报价单。
支持信息完整的单条报价，例如 Aud $50 Apple 3.35。
统一落入标准报价结构。
生成当前最高价行情墙。
保留历史价格切片，供后续错价检测使用。
解析失败、格式变化、缺上下文短报价进入异常区。
P1 暂不做：

AI 自动解析主链路。
每行人工审核。
供应商机器人自动报价。
缺价后自动去客人群问价。
自动识别漏单、错群、重复加单。
同事整理报价单参与最高价排名。
3. 核心用户场景
同事每天开始工作或收到供应商询价时，可以在报价墙搜索：

5. 解析策略
MVP 使用“客人群模板 + 通用字典”的方式，不做万能解析器。

通用字典负责标准化：

Steam / S team / 蒸汽 -> Steam
Razer / 雷蛇 / Razer Gold -> Razer
Xbox / X box -> Xbox
Apple / A pple -> Apple

US / USA / USD / 美金 / 美国 -> USD
UK / GBP / 英镑 -> GBP
CAD / 加拿大 / 加元 -> CAD
AUD / 澳大利亚 / 澳元 -> AUD
客人群模板负责解析该群稳定格式：

客人1 -> Apple 横白卡 / 竖卡 / 电子 / 代码修正规则模板
客人2 -> 简单卡种板块 + 国家价格模板
客人3 -> 多卡种长报价单板块模板
高置信解析结果自动入行情墙。

进入异常区的情况：

模板不匹配。
新卡种或新国家无法识别。
缺卡种短回复，例如 uk 10 / 6.25。
价格变化异常大。
行内容是 问、不拿、待定、不加账 等非直接价格。
条件作用范围不明确。

7. P2 预留
P1 必须保留价格历史切片，因为 P2 要用交易发生时间回查当时价格。

P2 第一刀是“价格加错”：

交易发生时间：14:03
交易入账价：5.10
当时价格墙最高价：5.30
系统提示：疑似价格加错，差额 0.20
后续再逐步做：

重复加单 / 多加单
客户加了、供应商没加
漏加单
账单加错群
双边都记错但各自公式成立
8. 验收标准
MVP 验收以真实样例为准：

客人 1 的 Apple 报价能解析出横白卡、竖卡、电子、代码等条件价格。
客人 2 的 Steam / Razer 简单国家价格能进入行情墙。
客人 3 的 Xbox / Razer / Steam / Google / PSN 等主要板块能解析出标准报价。
同一卡种同条件新价格会覆盖旧价格，旧价格保留历史。
新卡种会新增，不会清空旧报价。
缺上下文短回复不会自动入墙。
行情墙能按同卡最高价展示。
每条有效报价能追溯回原始消息和原文。
9. 默认假设
P1 事实源是客人原始报价，不是同事整理后的报价单。
一个客人群就是一个报价来源。
报价生效时间默认使用消息时间。
P1 主链路不用 AI。
AI 后续只做异常解释、模板辅助，不参与日常自动入墙。
供应商机器人自动问价放到后续阶段。
