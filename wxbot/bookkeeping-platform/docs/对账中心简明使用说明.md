# 对账中心简明使用说明

适用页面：`/reconciliation`

这页不是看利润总表的，它是给财务逐笔核账用的。

现在这版主要解决 4 件事：

1. 看每一笔卡账和每一笔财务加账
2. 找出汇率、RMB 公式、人工修改这类异常
3. 先按组合或组号汇总，再导出 CSV 去核对
4. 把额外手续费、补差、调账录成“逐笔财务加账”

---

## 页面上每一块是干什么的

### 1. 顶部 4 张卡片

- `未对账笔数`：当前筛选范围内，客户/供应商口径下还没完成对账的笔数
- `汇率公式异常`：非 RMB 交易里，金额 × 汇率 和 RMB 结果对不上的笔数
- `缺失汇率`：非 RMB 交易没填汇率
- `已修改未复核`：后台人工改过，但还需要复核的笔数

如果这些数字不为 0，说明这个范围里有问题要查。

### 2. 筛选区

- `范围`
  - `实时`：看当前还没关账的实时流水
  - `指定账期`：看某一个已经关掉的账期
  - `日期区间`：按日期范围查并导出
- `组合`
  - 选一个已经保存好的组号组合，例如 `2+3+4`
  - 适合看一整个财务口径，不要只看单个群
- `组号`
  - 如果不选组合，也可以直接看某一个组号，例如 `组 2`
- `群组 Key`
  - 这是下钻条件
  - 只有在你已经按组合或组号筛完后，想继续看某一个具体群时再用
- `角色`：一般看 `customer` 或 `vendor`
- `卡种`：只看某一种卡，比如 `xb`
- `编辑状态`
  - `全部`
  - `仅已编辑`
  - `仅未编辑`
- `异常类型`
  - `全部`
  - `未对账`
  - `汇率公式异常`
  - `缺失汇率`
  - `已修改未复核`

点 `刷新台账` 后，下面表格会按条件重载。

### 3. 导出按钮

现在有两个导出按钮：

- `导出逐笔明细 CSV`
- `导出汇总 CSV`

两个导出都会沿用你当前筛选结果，不需要重新筛一遍。

区别是：

- `逐笔明细`：每一笔都导出来
- `汇总`：按当前组合或组号整体汇总，不再拆到单个群

适合这几种场景：

- 发给财务做逐笔核对：用 `逐笔明细`
- 拿去跟 Excel、银行流水、供应商报表做总体对账：用 `汇总`
- 把异常筛出来单独处理：先筛异常，再导 `逐笔明细`

### 4. 财务加账表单

这块是给财务补录“系统原始交易之外的金额”。

常见例子：

- 手续费
- 补差
- RMB 加减
- 某一笔交易的额外修正

主要字段怎么填：

- `账期 ID`
  - 如果是已经关账后的补录，填对应账期
  - 如果是实时补录，可以留空
- `群组 Key`
  - 必填，例如 `wechat:g-100`
- `业务角色`
  - 一般填 `customer` 或 `vendor`
- `卡种`
  - 例如 `fee`、`adjustment`、`xb`
- `USD 金额`
  - 没有就填 `0`
- `汇率`
  - 没有就留空
- `RMB 金额`
  - 必填，正负按实际财务口径录入
- `关联交易 ID`
  - 如果这笔加账是挂在某条交易上的，就填那条交易 ID
- `说明`
  - 必填，写清楚原因，后面导出时才能看懂

点 `新增财务加账` 后，这笔会进入台账和导出结果。

### 5. 台账表格

表格里会同时显示两类数据：

- `transaction`：系统识别出来的原始交易
- `finance_adjustment`：财务手工补录的逐笔加账

重点看这些列：

- `群组`
- `角色`
- `卡种`
- `金额 / 汇率 / RMB`
- `异常标记`
- `编辑人 / 编辑时间`
- `说明`

如果某一行右侧有 `引用` 按钮，点它可以把这条交易的 `group_key / card_type / linked_transaction_id` 自动带到下方的财务加账表单里。

## 最常见的 3 种用法

### 用法 1：看今天实时有没有录错

1. 打开 `/reconciliation`
2. `范围` 选 `实时`
3. 如果财务是按一组群来看，先选 `组合` 或 `组号`
4. `异常类型` 先选 `汇率公式异常`
5. 看哪些行标红，再去核对
6. 再切到 `缺失汇率` 和 `已修改未复核` 分别看一遍

### 用法 2：核一个已经关账的账期

1. `范围` 选 `指定账期`
2. 选择要核的 `账期 ID`
3. 再选要核的 `组合` 或 `组号`
4. 再按 `customer`、`vendor` 分别筛
5. 要看逐笔就点 `导出逐笔明细 CSV`
6. 要给财务看整体小计就点 `导出汇总 CSV`

### 用法 3：补一笔手续费或补差

1. 如果这笔是针对某条交易的，先在台账里找到那行
2. 点 `引用`
3. 检查自动带出来的 `群组 Key / 卡种 / 关联交易 ID`
4. 填 `RMB 金额` 和 `说明`
5. 提交后刷新台账确认是否已入账

---

## 异常标记是什么意思

- `pending_reconciliation`
  - 这笔还在对账范围内，后面需要和 B 面数据核
- `rate_formula_error`
  - 金额、汇率、RMB 三者算不平
- `missing_rate`
  - 非 RMB 交易缺汇率
- `edited_unreviewed`
  - 后台有人改过这条记录，但还没人工复核

## 这版先做到哪里

这版已经有：

- 逐笔台账
- 组合筛选
- 组号筛选
- 财务加账
- 异常识别
- 明细 / 汇总 两种 CSV 导出

这版还没有：

- Excel/CSV 对方账单导入
- A/B 自动匹配
- 对账结果人工确认流

也就是说，现在能先把 A 面台账做扎实，方便财务导出去和 B 面核；自动对账会放到下一步。

---

## 给财务的最短操作建议

每天只做这 4 步就够：

1. 打开 `/reconciliation`
2. 先选今天要看的 `组合` 或 `组号`
3. 看顶部异常卡片有没有非 0，再把异常类型逐个筛一遍
4. 需要发别人核对时，按场景导出 `逐笔明细` 或 `汇总`

如果还有系统外金额，就在下面加一笔“财务加账”，不要只记在口头或 Excel 里。
