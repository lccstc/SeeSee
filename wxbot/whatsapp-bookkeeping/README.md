# WhatsApp Bookkeeping

独立的 WhatsApp 群组记账机器人，零 AI 开销，完全基于规则的消息处理。

## 功能特性

- ✅ 交易记录：支持多种格式（`+100rmb`、`-50rg 5.3` 等）
- ✅ 余额查询：`/bal`（显示未扎帐刀数概要）
- ✅ 交易历史：`/history`
- ✅ 类目明细：`/mx`（按类目 + 汇率统计刀数，分组 2 显示双币种）
- ✅ 扎帐功能：`/js`（结算未扎帐交易，不清空余额）
- ✅ 一键扎帐：`/alljs`（对所有有未扎帐账单的群组批量扎帐）
- ✅ 扎帐历史：`/settlements`
- ✅ 群组分组：`/set`（将群组分配到 0-9 分组）
- ✅ 群发消息：`/diy`（向指定分组的所有群组发消息，支持长文本）
- ✅ NGN 汇率：`/ngn`（设置奈拉汇率，仅分组 2 显示）
- ✅ 撤销交易：`/undo`
- ✅ 白名单管理：`/adduser`、`/rmuser`、`/users`
- ✅ 用户绑定：`/bind`
- ✅ 数据导出：`/export`
- ✅ 实时余额：交易后自动显示当前余额和 NGN 汇率
- ✅ 汇率验证：汇率超过 10 时报错提示
- ✅ 分组 2 专属：人民币 `¥` + 奈拉 `₦` 双币种显示（🥛₦ 格式）
- ✅ NGN 自动计算：`数字*数字` 格式（如 `100*5.3`）
- ✅ 多管理员支持：支持配置多个管理员账号
- ✅ 倒计时提醒：交易后设置付款提醒（`XXmins` 格式）

## 配置

编辑 `config.json`：

```json
{
  "whatsapp": {
    "authDir": "./auth"
  },
  "masterPhone": "+84389225210",
  "masterPhones": ["+84389225210", "+85257006866"],
  "logLevel": "info",
  "sync": {
    "enabled": false,
    "endpoint": "https://ledger.example.com/api/sync/events",
    "token": "replace-with-sync-token",
    "sourceMachine": "wa-node-01",
    "flushIntervalMs": 1000,
    "batchSize": 20,
    "requestTimeoutMs": 5000
  }
}
```

**配置说明**：
- `masterPhone`: 主管理员账号（兼容旧配置）
- `masterPhones`: 多管理员配置（数组，优先级高于 `masterPhone`）
- `sync.enabled`: 是否启用实时同步到总库
- `sync.endpoint`: 总库对外暴露的 `POST /api/sync/events` 地址
- `sync.token`: 总库接收接口的 Bearer Token
- `sync.sourceMachine`: 入口机器标识，建议写固定机器名
- `sync.flushIntervalMs`: 同步 worker 轮询间隔，默认 1000ms
- `sync.batchSize`: 单次最多推送多少条事件
- `sync.requestTimeoutMs`: 单次请求超时毫秒数

## 实时同步到总库

当前已支持把 WhatsApp 账面动作异步推送到总库：

- 交易新增
- `/undo` 撤销
- `/clear` 清空
- `/set` 分组设置
- `/js`、`/alljs` 扎账

同步语义：

- 先写本地 SQLite，再写本地 `sync_outbox`
- 后台 worker 异步推送到总库
- 网络失败不会影响本地记账成功
- 成功后事件标记为 `sent`

### 最小配置示例

```json
{
  "sync": {
    "enabled": true,
    "endpoint": "https://your-domain.com/api/sync/events",
    "token": "your-sync-token",
    "sourceMachine": "wa-node-01",
    "flushIntervalMs": 1000,
    "batchSize": 20,
    "requestTimeoutMs": 5000
  }
}
```

### 真机烟测步骤

1. 先在总库机器启动 `bookkeeping-platform`，并配置 `BOOKKEEPING_SYNC_TOKEN`
2. 在 WhatsApp 机器把 `sync.enabled` 改成 `true`
3. 启动机器人并保持原有登录态
4. 在测试群发送一笔最简单的账单，例如 `rmb+100`
5. 检查：
   - 本地 `transactions` 已写入
   - 本地 `sync_outbox` 有对应事件并最终变为 `sent`
   - 总库看板出现对应群余额
6. 再发 `/undo` 和 `/js`，确认删除和扎账也能同步

## 交易格式

支持多种输入格式：

| 格式 | 示例 | 说明 |
|------|------|------|
| 符号在前 | `+100rmb` | 收入 100 元 |
| 符号在前 | `-50rg 5.3` | 支出 50 RG × 5.3 |
| 符号在后 | `100rmb+` | 收入 100 元 |
| 符号在后 | `25rg-` | 支出 25 RG |
| 类别在前 | `rmb+100` | 收入 100 元 |
| 类别在前 | `rg-25 5.3` | 支出 25 RG × 5.3 |

### 类别说明

- **rmb**: 人民币（正数类别）
- **rg/sp/gs/xb/it/st/ft/mx/gg/dg/ae/lulu/dk/ebay/rb**: 负数类别（符号反转）
- **uber/xc/cvs/ymx/psn/chime/ks/tt/nike/nd/cash/ps**: 新增负数类别（符号反转）

### 汇率说明

- 非 RMB 交易必须提供汇率
- 汇率必须在 0-10 之间，超过会报错
- 示例：`+25rg 4.0` = 25 刀 × 4.0 = 100 元

## 明细说明

`/mx` 命令按类目和汇率分组统计，显示刀数和人民币/奈拉：

### 普通分组
```
📊 明细:
RG:
  ×3: 200 = -600.00
  ×4: 80 = -320.00
RMB:
  500.00
```

### 分组 2（双币种）
```
📊 明细:
💰 余额：-2860.00
🥛₦ 195
IT:
  ×4.9: 400 = -¥1960.00 (-₦382200)
  ×5: 100 = -¥500.00 (-₦97500)
RG:
  ×5: 25 = -¥125.00 (-₦24375)
```

### /bal 命令（分组 2）
```
📊 余额：-2860.00
📋 群组：分组 2
📝 未扎帐：5 笔

明细:
IT:
  ×4.9: 400
  ×5: 100
RG:
  ×5: 25

🥛₦ 195
```

- 刀数自动对冲（正负相抵）
- 合计显示净刀数和总人民币/奈拉

## 扎帐说明

`/js` 命令用于结算自上次扎帐以来的所有未结算交易：

- 不会清空余额
- 只是标记交易为"已扎帐"状态
- 生成扎帐汇总（按类目统计）
- 白名单用户都可以扎帐

扎帐后 `/bal` 和 `/mx` 显示"所有交易已扎帐"。

## 群组分组说明

**只有分配到数字（0-9）的群组才能激活记账功能**。

### 设置分组

```
/set 0    # 将本群分配到分组 0
/set 1    # 将本群分配到分组 1
...
/set 9    # 将本群分配到分组 9
```

- 只有管理员可以设置分组
- 每个群只能分配到一个数字
- 未分配数字的群组无法使用记账功能

### 群发消息

```
/diy 1 大家好，这是通知内容
/diy 2 明天涨价了
```

- 向指定分组的所有群组发送自定义消息
- 支持多行长文本（如价格表）
- 白名单用户可以发送
- 自动批量发送（每 5 群暂停 1.5 秒，防限流）
- 失败自动重试（最多 3 次）
- 发送完成后显示详细报告

## NGN 自动计算功能

分组 2 支持快速 NGN 计算模式，输入 `数字*数字` 格式：

```
100*5.3    # 计算 100 × (5.3 × NGN汇率) = ₦结果
```

**计算逻辑**：
- 第一个数：数量
- 第二个数：中间价
- 结果 = 第一个数 × (第二个数 × 当前 NGN 汇率)

**示例**：
```
输入：100*5.3
NGN汇率：195
输出：100*1031.50=₦103150
```

**权限**：仅分组 2 的白名单用户可用

## 倒计时提醒功能

交易后可设置付款提醒，适用于赊账场景：

```
+500 IT ×5    # 先记录交易
40mins        # 设置 40 分钟后提醒付款
```

**使用说明**：
1. 先发送交易记录（如 `+500 IT ×5`）
2. 在 **5 分钟内** 发送 `XXmins`（如 `40mins`）
3. 倒计时结束后，机器人自动在群内发送付款提醒

**提醒格式**：
```
⏰ Payment Reminder!
📋 +500 IT ×5
⏱️ 40 minutes elapsed
```

**注意事项**：
- 必须在交易后 5 分钟内设置倒计时
- 倒计时时间到后自动发送提醒到群组
- 程序重启后，未触发的倒计时会自动恢复
- 支持多个倒计时同时进行

## NGN 汇率说明

```
/ngn 194.2    # 设置 NGN 汇率为 194.2
```

- 设置后，NGN 汇率会显示在所有余额信息的下方
- 显示格式：`🥛₦ 194.2`（仅分组 2 显示）
- 白名单用户可以设置
- 显示位置：
  - `/bal` 命令输出中（仅分组 2）
  - `/mx` 命令输出中（仅分组 2）
  - 交易后实时反馈中（仅分组 2）

## 分组 2 双币种说明

### 货币符号
- **人民币 (CNY)**: `¥`
- **尼日利亚奈拉 (NGN)**: `₦`

### 交易后反馈（分组 2）
```
✅ +25 RG ×5 = -¥125.00 (-₦24275)
📊 余额：+100.00
🥛₦ 194.2
```

### 汇率记录逻辑
- 每笔交易记录**交易发生时**的 NGN 汇率
- 账单显示使用**交易时的汇率**，不受后续汇率变更影响
- 10 点汇率 194 时的交易永远显示 194 的 NGN 金额
- 12 点汇率改为 196 后，新交易显示 196，老交易仍显示 194
