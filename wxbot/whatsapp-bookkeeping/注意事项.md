# WhatsApp 机器人注意事项

## 问题记录：Connection Failure 连接失败

### 发生时间
2026-02-24

### 问题现象
运行 `npm start` 后出现 `Connection Failure` 错误，无法显示二维码，机器人无法连接 WhatsApp。

错误信息示例：
```
{"level":30,"time":...,"trace":"Error: Connection Failure
    at WebSocketClient.<anonymous> (file:///.../@whiskeysockets/baileys/lib/Socket/socket.js:785:13)
    ...","msg":"connection errored"}
```

### 根本原因

#### 1. 缺少必要的 Baileys 配置参数

**问题代码（错误）：**
```typescript
this.socket = makeWASocket({
  auth: {
    creds: state.creds,
    keys: makeCacheableSignalKeyStore(state.keys, this.logger),
  },
  logger: this.logger,
  browser: Browsers.macOS("Chrome"),
  markOnlineOnConnect: true,
});
```

**正确代码：**
```typescript
// 1. 先获取最新 WhatsApp Web 版本
const { version } = await fetchLatestBaileysVersion();

this.socket = makeWASocket({
  auth: {
    creds: state.creds,
    keys: makeCacheableSignalKeyStore(state.keys, this.logger),
  },
  version,              // ✅ 必须：指定 WhatsApp Web 版本
  logger: this.logger,
  browser: Browsers.ubuntu("Chrome"),  // ✅ 建议：使用 ubuntu 标识
  markOnlineOnConnect: true,
  syncFullHistory: false, // ✅ 必须：禁用全历史同步
});
```

#### 2. 关键参数说明

| 参数 | 作用 | 必要性 |
|------|------|--------|
| `version` | 指定 WhatsApp Web 版本号，通过 `fetchLatestBaileysVersion()` 获取 | ⭐⭐⭐ 必须 |
| `syncFullHistory` | 是否同步完整聊天记录，设为 `false` 避免连接问题 | ⭐⭐⭐ 必须 |
| `browser` | 浏览器标识，建议使用 `Browsers.ubuntu("Chrome")` | ⭐⭐ 建议 |

### 解决方法

#### 步骤 1：导入必要函数
```typescript
import makeWASocket, {
  DisconnectReason,
  useMultiFileAuthState,
  WASocket,
  makeCacheableSignalKeyStore,
  Browsers,
  fetchLatestBaileysVersion,  // 新增导入
} from "@whiskeysockets/baileys";
```

#### 步骤 2：在 connect() 方法中获取版本
```typescript
async connect(): Promise<void> {
  const { state, saveCreds } = await useMultiFileAuthState(this.config.authDir);

  // 获取最新 WhatsApp Web 版本（关键！）
  const { version } = await fetchLatestBaileysVersion();

  this.socket = makeWASocket({
    auth: {
      creds: state.creds,
      keys: makeCacheableSignalKeyStore(state.keys, this.logger),
    },
    version,              // 使用获取到的版本
    logger: this.logger,
    browser: Browsers.ubuntu("Chrome"),
    markOnlineOnConnect: true,
    syncFullHistory: false, // 禁用全历史同步
  });

  // ... 后续代码
}
```

#### 步骤 3：重新编译运行
```bash
npm run build
npm start
```

## 其他常见问题

### 1. 认证文件丢失（auth 目录为空）

**现象：** 之前能正常登录，突然需要重新扫码

**原因：** `auth/` 目录存放 WhatsApp 认证信息，如果被删除或清空，需要重新登录

**解决方法：**
- 情况 A：重新扫码登录（推荐）
  ```bash
  # 清空 auth 目录
  rm -rf auth/*
  # 重新运行，扫描二维码
  npm start
  ```

- 情况 B：从备份恢复
  ```bash
  # 从 OpenClaw 复制认证文件（如果之前用过）
  cp -r ~/.openclaw/credentials/whatsapp/default/* ./auth/
  ```

### 2. 代码修改后未重新编译

**现象：** 修改了 `src/whatsapp.ts` 但运行结果没有变化

**原因：** TypeScript 需要先编译成 JavaScript 才能运行

**解决方法：**
```bash
# 每次修改 src/ 下的文件后，都要重新编译
npm run build

# 然后再运行
npm start

# 或者使用开发模式（自动编译）
npm run dev
```

### 3. 依赖问题

**现象：** 各种奇怪的模块错误

**解决方法：**
```bash
# 删除 node_modules 重新安装
rm -rf node_modules package-lock.json
npm install
npm run build
```

## 开发 checklist

修改代码后，按以下顺序操作：

1. ✅ 修改 `src/` 下的 TypeScript 源文件
2. ✅ 运行 `npm run build` 编译
3. ✅ 运行 `npm start` 启动
4. ✅ 如出现连接问题，检查：
   - `version` 参数是否已设置
   - `syncFullHistory: false` 是否已设置
   - `auth/` 目录权限是否正常
