| Boundary | Description |
|----------|-------------|
| parser output -> candidate persistence | Untrusted parse results become durable DB records here. |
| PostgreSQL schema -> runtime startup | Missing candidate columns or tables would silently weaken custody if startup does not fail. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | T | `quote_documents` / `quote_candidate_rows` schema | mitigate | Add dedicated candidate fields and child table so candidate intent never reuses `quote_price_rows` semantics. |
| T-01-02 | R | Candidate persistence evidence | mitigate | Require raw message, sender, group, message time, source line, parser lineage, and JSON evidence fields in the contract and backend tests. |
| T-01-03 | D | Runtime startup on partial schema | mitigate | Extend `BookkeepingDB._verify_schema()` so missing candidate columns or `quote_candidate_rows` block startup with an explicit mismatch error. |
</threat_model>
