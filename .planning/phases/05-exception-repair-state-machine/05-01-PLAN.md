| T-05-01-02 | R | repair-case lineage | mitigate | Persist origin exception/document/validation IDs plus timestamps and frozen profile snapshots so later remediation can prove which evidence was packaged. |
| T-05-01-03 | D | runtime failure packaging | mitigate | Make repair-case creation idempotent per exception row so repeated runtime or replay passes cannot explode duplicate cases for the same failure event. |
| T-05-01-04 | E | repair-case workflow | mitigate | Explicitly forbid any write to `quote_price_rows` or guarded publisher paths in this plan; repair packaging stays evidence-only until Phase 03. |
