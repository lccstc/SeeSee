# Plan 02.1-03 Summary

## Outcome

Wave 3 turned the approved top-volume fixtures into deterministic bootstrap profile payloads and proved those bootstraps stay inside candidate-only custody.

## Delivered

- Added `scripts/bootstrap_quote_group_profiles.py` to generate `group-parser-v1` profile payloads from approved fixtures only.
- Added fixture alias support for the planned top-volume groups while still failing fast when a requested fixture is not approved in `approved_top8.json`.
- Added `build_group_parser_template_from_gold_fixture(...)` to `template_engine.py` so bootstrap templates are derived from raw-message line order rather than parser guesses.
- Added `tests/test_bootstrap_quote_group_profiles.py` for generator shape, alias coverage, allowlist enforcement, and zero DB writes.
- Added runtime regressions proving bootstrapped top groups persist candidate rows and validation runs without mutating `quote_price_rows`.
- Added web replay regression proving a bootstrapped replay stays candidate-only with `mutated_active_facts = false`.

## Business Effect

- High-frequency missing-template groups now have a deterministic “seed the profile, then replay” path instead of waiting for manual one-off rescue every time.
- Unapproved or unbootstrapped groups still fall back to the exception path; nothing here created a publish shortcut.
